1.4 HTTP（HyperText Transfer Protocol）

1.4.1 工作过程：

  1. 客户端连接到 Web Server
  2. 发送 HTTP 请求
  3. 服务器接收到请求并返回 HTTP 响应
  4. 释放 TCP 连接
  5. 客户端解析 HTML 内容

1.4.2 HTTP Request Header

请求行（头部行）：请求方法，URL，HTTP 版本

1.4.3 HTTP Response Header

协议版本，状态码，原因短语 content-length: content-type: text/html

1.4.4 常见的状态码

  1. 200
  2. 301
  3. 400
  4. 500
  5. 502

1.4.5 URL vs URI

URL(Uniform Resource Locator，统一资源定位符) URI(Uniform Resource Identifier，统一资源标识符)，用于标识互联网资源名称的字符串，常见形式就是 URL，另外还有一个叫 URN(Uniform Resource Name，统一资源名称)，通俗地说，URL 和 URN 是 URI 的子集，URI 属于更高层次的抽象。

1.4.6 Web 工作原理

> 当用户打开浏览器，输入一个 URL，按下回车键，请大致描述发生的所有事件。

1.5 HTML5 & CSS3

1.5.1 HTML5

  1. 新变化

    ```html <!– 声明 –> <!DOCTYPE html> <!– 编码 –> <meta charset="UTF-8"> <!– 具有 boolean 属性 –> <input type="text" disabled="false" readonly="true" /> <!– 新增结构元素、表单属性、多媒体属性、其他元素 –> <header></header> <section></section> <aside></aside> <footer><footer> ```

  2. 高级功能：
    • 绘制图形，如签名
    • 本地存储和离线应用：本地存储分为 Web Storage(原 HTML 4 中扩展)和本地 DB(HTML5 新增)
    • 通信 API：两种方式：跨文档消息传输和 Web Sockets API。解决了同源通信和跨域通信，同时 Server 不再需要被动的等待 Client 发出请求，只要两端建立连接后，Server 端就可以主动推送数据给 Client 端。
    • 处理线程：在 HTML4 和 JS 编写程序时，所有的处理都是单线程内执行，话费时间比较长。HTML5 新增 Web Workers API，可以容易地在后台创建线程，并且这个后台线程还能创建多个子线程。
    • 获取地理位置
